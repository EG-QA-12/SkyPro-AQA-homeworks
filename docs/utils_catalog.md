# Каталог утилит проекта

Модуль `framework/utils/` содержит набор небольших, но полезных функций и классов, которые используются в тестах и Page Objects. Ниже приводится краткое описание каждого файла.

| Файл | Назначение |
|------|------------|
| `auth_utils.py` | Высокоуровневые операции авторизации и управление токенами |
| `cookie_constants.py` | Константы и схемы валидации для файлов куков |
| `cookie_helper.py` | Функции чтения, записи и обновления cookies в формате JSON |
| `gui_helpers.py` | Утилиты для вывода статуса в Tkinter/ttk GUI |
| `reporting/allure_utils.py` | Вспомогательные функции для логирования шагов и вложений в Allure |
| `secure_auth_utils.py` | Безопасная работа с паролями и переменными окружения |
| `url_utils.py` | Формирование URL-адресов и добавление параметров сессии |

## Общие рекомендации

1. **Единый источник правды**: не дублируйте логику — вызывайте helper, если он уже реализован.
2. **Малые, переиспользуемые функции**: каждая функция должна решать одну задачу.
3. **Именование**: используйте говорящие имена, чтобы по коду было ясно, что делает функция.
4. **Документирование**: добавляйте type hints и docstrings — это облегчает автогенерацию документации.
5. **Тесты**: для каждой нетривиальной функции добавляйте unit-тест в `tests/unit/`.
