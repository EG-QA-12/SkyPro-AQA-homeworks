# Глоссарий терминов BLL_tests

Этот документ содержит ключевые термины, используемые в проекте, с объяснениями и примерами. Полезно для быстрого понимания концепций.

## Основные термины

### Fixture
Переиспользуемая функция в pytest, которая готовит данные или окружение для тестов. Пример: `auth_user` создаёт тестового пользователя.

```python
@pytest.fixture
def auth_user():
    return create_user()
```

### Page Object
Класс, представляющий страницу веб-приложения. Скрывает детали (локаторы) и предоставляет методы действий. Пример: `LoginPage` для формы входа.

### Allure
Инструмент для красивых отчётов тестов. Добавляет шаги, attachments и категории.

### Cookie
Файл сессии для авторизации. В проекте хранятся в `cookies/`, используются для быстрого входа без ввода логина/пароля.

### End-to-End (E2E) тест
Полный сценарий от начала до конца, имитирующий пользователя. Располагается в `tests/e2e/`.

## Дополнительные термины

### Integration тест
Проверка взаимодействия компонентов (БД + API). В `tests/integration/`.

### Unit тест
Проверка отдельной функции без зависимостей. В `tests/unit/`.
