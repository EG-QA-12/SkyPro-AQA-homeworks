# План рефакторинга структуры репозитория

> Ветка: `refactor/clean-structure`
>
> Цель: упростить и стандартизировать расположение файлов, убрать дубликаты, сделать проект прозрачным для новых участников команды.

## 1. Текущее состояние
Кратко:
* Дублирование утилит и служебных папок.
* Вложенный проект `projects/auth_management/` содержит артефакты (`logs/`, `reports/`, `venv/`).
* В репозиторий попадают результаты тестов (`allure-results/`, `screenshots/`, `logs/`).
* Новичку тяжело ориентироваться.

## 2. Целевая структура (укороченная)
```
Bll_tests/
├── framework/          # Общие инструменты
│   ├── utils/          # auth_utils, allure_utils, cookie_helper и др.
│   ├── fixtures/
│   └── app/            # Page Objects общего назначения
├── projects/
│   └── auth_management/
│       ├── pages/      # Специфичные Page Objects
│       └── tests/      # Тесты домена
├── tests/
│   ├── e2e/
│   └── integration/
├── docs/
├── scripts/
├── config/
└── .gitignore          # Расширен для артефактов
```

## 3. Дорожная карта изменений
| Этап | Действие |
|------|----------|
| 0 | Создать ветку `refactor/clean-structure` *(выполнено)* |
| 1 | Создать этот план в `docs/structure_refactor_plan.md` *(выполнено)* |
| 2 | Переместить `projects/helpers/*` → `framework/utils/` |
| 3 | Переместить полезное из `tests/e2e/utils/` → `framework/utils/` |
| 4 | Обновить импорты во всех файлах (`from projects.helpers` → `from framework.utils`) |
| 5 | Почистить `projects/auth_management/` (убрать `logs/`, `reports/`, `venv/`) |
| 6 | Расширить `.gitignore` для артефактов |
| 7 | Запустить тесты и убедиться, что всё зелёное |
| 8 | Обновить корневой `README.md` с новой схемой |

## 4. Правила коммитов
* `docs:` – документация
* `refactor:` – перемещение/переименование без изменения логики
* `fix:` – исправление импорта, тестов, конфигурации

## 5. Откат
Каждый этап – отдельный коммит. В случае ошибки можно выполнить:
```
git reset --hard <хеш предыдущего коммита>
```
