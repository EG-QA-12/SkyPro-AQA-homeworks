# Конфигурация End-to-End тестов

## Описание

Этот модуль содержит конфигурационные файлы и утилиты для настройки и управления end-to-end тестами.

## Структура

```
config/
├── __init__.py          # Инициализация модуля, экспорт конфига
├── e2e_config.py        # Основной файл конфигурации
└── README.md           # Эта документация
```

## Основные компоненты

### e2e_config.py

Основной конфигурационный файл, который:
- Настраивает пути к данным пользователей
- Инициализирует базовую конфигурацию
- Управляет переменными окружения

### Использование

```python
# Импорт конфигурации
from tests.e2e.config import config

# Использование настроек
print(f"Путь к логам: {config.LOG_PATH}")
print(f"Путь к кукам: {config.COOKIES_PATH}")
```

## Настройка путей

- `LOG_PATH`: `auth_project/user_data/app.log`
- `COOKIES_PATH`: `auth_project/user_data/cookies.json`

## Автоматическая инициализация

При импорте модуля автоматически:
1. Определяются корректные пути к директориям
2. Добавляется корневая директория проекта в `sys.path`
3. Загружается базовая конфигурация из `src.config`
4. Устанавливаются переменные окружения

## Обновление конфигурации

При необходимости изменить конфигурацию:
1. Отредактируйте `e2e_config.py`
2. Обновите соответствующие переменные окружения
3. Убедитесь, что все зависимые тесты работают корректно

## Замечания по безопасности

- Не храните чувствительные данные в конфигурационных файлах
- Используйте переменные окружения для секретов
- Проверяйте права доступа к файлам конфигурации
