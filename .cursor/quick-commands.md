# Быстрые команды для работы с ролями

## Переключение ролей:
```
/role aqa          # Ведущий AQA - архитектурные задачи
/role sdet         # SDET Mentor Agent - автономная разработка
/role mentor       # Python Automation Mentor - обучение
```

## Часто используемые команды по ролям:

### Ведущий AQA (/role aqa):
```
/analyze code      # Анализ кода на соответствие принципам
/refactor          # Рефакторинг с объяснением
/architecture      # Архитектурные решения
/performance       # Анализ производительности
/best-practices    # Внедрение лучших практик
```

### Python Automation Mentor (/role mentor):
```
/explain concept   # Объяснение концепции
/practice task     # Практическое задание
/interview prep    # Подготовка к собеседованию
/compare tools     # Сравнение инструментов
/learning path     # План обучения
```

### SDET Mentor Agent (/role sdet):
```
/analyze task      # Анализ задачи и аудит кодовой базы
/plan development   # Планирование с оценкой рисков
/execute step      # Пошаговое выполнение с контролем
/git operations    # Управление Git-операциями
/risk assessment   # Оценка рисков (wcgw)
```

## Контекстные команды:
```
/help roles        # Справка по ролям
/current role      # Показать текущую роль
/switch to [роль]  # Альтернативный способ переключения
```

## Примеры использования:

1. **Создание нового теста**:
   ```
   /role aqa
   Создай тест для проверки авторизации через API
   ```

2. **Объяснение концепции**:
   ```
   /role mentor
   Объясни принцип Page Object Model
   ```

3. **Анализ существующего кода**:
   ```
   /role aqa
   Проанализируй этот тест на соответствие принципам
   ```

4. **Полный цикл разработки**:
   ```
   /role sdet
   Создай новый тест для проверки авторизации с полным циклом разработки
   ```
